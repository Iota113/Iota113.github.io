CS2030S Programming Methodology II

Taken in 24/25 Sem 2
Profs: Boyd Anderson, Ashish Deepak Dandekar, Ooi Wei Tsang

This module teaches mainly Object Oriented Programming and Functional Programming on Java. We are also exposed to Vim and UNIX as all coding — problem sets, practical exams — are done on the terminal with Vim. I did enjoy using Java at the end although everyone likes to complain about it.

  1. Java OOP
  2. Heap and Stack, Types, Exception Handling, Java Generics
  3. Functional Programming - Immutability, Monads and Functors, Functional Interfaces, Streams
  4. Asynchronous Programming - Threads, CompletableFuture

A monad is a monoid in the category of endofunctors ~ I stole this line from some YouTube video.

It was quite reassuring to see Boyd Anderson again after 1101S. Boyd retains his usual teaching style, always using examples of him "failing" to implement something to motivate the introduction of new concepts. Ashish did some lectures on functional programming. He didn't employ the same theatrics as Boyd but he's solid too. Wei Tsang was just there to answer questions in chat and on Ed.

Assessment:

  • CA (10%)
  • PE0 (1%*) - 26/26
  • PE1 (15%) - 27.5/40
  • PE2 (25%) - 35/40
  • Midterm (20%) - 55/70
  • Final (30%) - 89/100
    • *1% awarded if you didn't get full marks for PE1 and PE2, which will be the case for most of you.

PE0 was a giveaway. PE1 is about applying OOP principles. Most people I know didn't finish PE1, that seems to be trend. Having said that, I know some people who got full marks because. PE2 is on making stream pipelines. Though this sem they had us implement some nested classes with generics too. Most people found PE2 manageable.

Midterm and Final are basically the same style as 1101S papers, just without the "problem-solving" aspect because that's saved for 2040S, probably. Majority MCQs with 1-2 short questions. No actual coding here really, besides reading code.

Something about type inference: you can think of subtyping (<:) as the less than or equal to relation (≤) from math, so the problem becomes solving inequalities. This (should) work since subtyping is a partial order relation. On that note, might also be useful to think of bounded wildcards in terms of sets hmm.

Overall, a less impressive and more boring sequel to 1101S / I am not very good at it.
